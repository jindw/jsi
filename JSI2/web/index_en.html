JSI Introduction:

JSI project in two parts: core framework and a series of sub-project based on the core framework.

Core framework designed to provide an open, non-invasive javascript library management solutions and a comprehensive front-end development, debugging, document parsing, template compiling, packaging and exporting environmental support.

It could serve as a developing time javascript management tools to allow developers to enjoy the various convenient during the development time; can also run as a runtime javascript framework;so that the author of the javascript library can prepare the dependence of the relevant javascript library, so that end-users from the cumbersome rely on the management of freed, improve the usability of the javascript library.

JSI has a sound mechanism for effective isolation module, facilitate the reuse, thirdparty javascript management, to avoid the naming of their pollution problems. At the same time strongly simple pursuit of freedom, not the introduction of special syntax, the javascript library in framework without relying on the framework.

JSI as an open javascript library management framework, any official built-in libraries will not provided, as far as possible achieve the neutrality of the javascript library. but I also will chair the JSIDE (JSI SIDE) the development of this project, it will be the basis of JSI , To do some practice, and share some of the optional library. At the same time other users according to their preferences can also integrate other javascript library, when we will provide some integration of some well-known javascript library as examples (such as: jQuery, Prototype, Scriptaculous, YAHOO UI).

The sub-projects of JSI, can provide a useful function of the specific. such as the page decorative framework.

Core Function:

  • load on demand 除常见的阻塞式装载方式，JSI还提供了无阻塞的按需装载方式（延迟同步装载，异步装载，JSI2.0+）
  • dependence management 脚本依赖的暴露、扩散是复杂度陡增的最大祸根。
    JSI提供了完善的依赖管理方案，将依赖终结在类库开发者手中。提高类库的易用性
  • module isolation 传统方式使用类库，将其直接导入进来，在全局上下文执行，这带来一个名称冲突的隐患；
    JSI中，每个脚本都有单独的执行上下文，不必担心不同脚本的命名冲突。
    甚至，在JSI的保护下，你可以是无忌惮的自由的使用内部元素。

Sub Project

  • Self Parsed Document 通过对项目脚本的实时解析，生成API文档；
    简单方便，保持文档与代码的同步。
  • Page Decorate Framework JSI组件模型是一种用来装饰简单html元素的框架，使用简单的xml标记，标识其装饰行为，比如将一个普通的input装饰成一个日期输入控件，将一个html ul标记装饰成菜单或树，将一个textarea装饰成一个代码语法高亮显示区域，或一个wysiwyg html编辑器。

    JSI启动后将自动检查decorator标记，构建层次结构，自动做相关类的寻找、导入和装饰操作；实现零脚本代码的web富客户端编程。
  • JSISide Template Engine 一个简单的模板引擎。
    JSDoc 及一些示例装饰器都基于此。
